```python
import pyzipper

# 获取密码
def getPassword(fileName: str):
    with open(fileName, "rb") as file:
        bytes = file.read()

    hexs = []
    texts = []
    for byte in bytes:
        hex = f"{byte:02x}"
        hexs.append(hex)
        char = chr(byte) if 32 <= byte <= 126 else "."
        texts.append(char)

    text = "".join(texts)
    return text.split("The password is ")[1]

# 解压函数
def unZip(fileName: str, password: str):
    with pyzipper.AESZipFile(fileName, 'r', encryption=pyzipper.WZ_AES) as zipFile:
        zipFile.setpassword(password.encode())
        zipFile.extractall(path='.')
        print(f"解压{fileName}成功({password})")

# 循环解压
for i in range(999, 0, -1):
    unZip(f"flagggg{i}.zip", getPassword(f"flagggg{i}.zip"))
```

```python
import hashlib

class StreamCipher:
    def __init__(self, key):
    
        self.S = list(range(256))
        self.i = 0
        self.j = 0
        
        j = 0
        key_bytes = self._key_to_bytes(key)
        for i in range(256):
            j = (j + self.S[i] + key_bytes[i % len(key_bytes)]) % 256
            self.S[i], self.S[j] = self.S[j], self.S[i]
    
    def _key_to_bytes(self, key):
        
        if isinstance(key, str):
            return hashlib.sha256(key.encode()).digest()
        elif isinstance(key, bytes):
            return hashlib.sha256(key).digest()
    
    def _prga(self):
        
        self.i = (self.i + 1) % 256
        self.j = (self.j + self.S[self.i]) % 256
        self.S[self.i], self.S[self.j] = self.S[self.j], self.S[self.i]

            data = data.encode('utf-8')

            key_byte = self._prga()
            result.append(byte ^ key_byte)
```

**分析：**

- 看到代码

  ```python
  ct1 = pow(msg, e, N)
  ct2 = pow(msg, p+q, N)
  ```

  它存在`共模攻击`其中`e`为`e1`，`p+q`为`e2`

- 然后`p+q`是两个相同`bit位`的超大素数和，我们已知`N = p * q`但是不可能分解它太大了，我们可以用近似法求`p+q ≈ 2√N`但这种方法还是有最大`2**(1024-1)`种情况需要枚举，最理想的情况是`p == q`或`p、q相差的绝对值小于等于1w`但这两种都属于是特殊情况了我们先待定

- 于是我们想到了`欧拉函数Φ(N) = phi_N = (p-1)*(q-1) = N-(p+q)+1`推出`p+q = n+1-Φ(N)`，然后根据`欧拉定理gcd(m,N) = 1`推出`m**ϕ(N) ≡ 1 (mod N)`和m**-ϕ(N) ≡ 1 (mod n)，由此得出

  ```python
  m**e ≡ ct1 mod N (e = 65537)
  m**(N+1) ≡ ct2 mod N
  ```

  同理共模

  ```python
  ct1 ≡ m**e mod N (e = 65537)
  ct2 ≡ m**(N+1) mod N
  ```

### ret2rop(xingluo)

#### 解题过程

**打开ida看看：**

```c
void __cdecl vuln()
{
  struct {char buf[32];char mask[32];} frame; // [rsp+0h] [rbp-50h] BYREF
  ssize_t n; // [rsp+40h] [rbp-10h]
  ssize_t i; // [rsp+48h] [rbp-8h]

  puts("please int your name");
  read(0, name, 0x10uLL);
  puts("please introduce yourself");
  getRandom(frame.mask, 32LL);
  n = read(0, &frame, 0x100uLL);
  if ( n > 0 )
  {
    for ( i = 0LL; i < n; ++i )
      frame.buf[i] ^= frame.mask[i];
  }
}
```

**分析：**

还找到了sytstem和/bin/sh(虽然不知道为什么/bin/sh没用...)，直接打算构造rop链。发现没有pop rdi，但可以用pop rsi和mov rdi rsi来替代。
但这里有一个异或，会把我们的rop链异或成奇怪的东西。那么我们就想想怎么绕过这个异或，用的是mask数组来异或，我们可以覆盖这个数组的值，但光覆盖这个数组的值是没办法绕过异或的，这是把后面的数据与前面的数据异或从而当作前面的数据，后门我们要构造rop链没法一直控制mask。那么我们就让i>n从而不让它执行异或来绕过就好了。这里有个问题，覆盖i还是n呢。我个人感觉其实都可以，控制数据的事而已。我选择覆盖前面的n，把他覆盖为0。n是我们输入的长度。那么我们构造rop链后看看长度就好了，我这里用print(hex(len(payload)))测得的长度为0x98。当i=64时到n。那么32+64=96，我们在96的位置输入p64(0x98)即可绕过，把n赋值为0。然后就是正常的构造rop了。注意这里那个/bin/sh是用不了的，我们要在前面那个name里面输入/bin/sh绕后把name的地址当/bin/sh的地址。

**exp如下：**

